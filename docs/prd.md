# 个人博客 PRD

## 1. 项目目标

做一个轻量、稳定、好维护的个人博客，用于发布和沉淀个人文章。

第一版优先保证：

- 写作和发布流程简单
- 页面加载快
- 阅读体验好
- SEO 基础完整
- 能展示个人信息与个人定位
- 为后续作品集展示预留扩展空间
- 不引入不必要的后端复杂度

## 2. MVP 范围

### 必做

- 首页
- 文章列表页
- 文章详情页
- 标签/分类
- 归档页
- 关于页
- 作品集预留入口
- RSS
- sitemap.xml
- 轻量站内搜索
- 基础 SEO 信息
- 响应式阅读体验

### 暂不做

- 评论
- 点赞
- 阅读量展示
- 用户登录
- 投稿
- 后台/CMS
- 完整作品集管理系统
- 邮件订阅
- 服务端搜索

## 3. 内容管理

文章由站长在代码仓库中用 Markdown 编写。

推荐结构：

```txt
content/posts/article-slug/
  index.md
  cover.jpg
```

每篇公开文章必须包含 frontmatter：

```yaml
---
title: "文章标题"
description: "文章摘要"
date: "2026-06-26"
tags: ["标签"]
category: "分类"
slug: "article-slug"
draft: false
---
```

规则：

- `description` 必须手写，不自动截取正文。
- `slug` 使用小写英文、数字和连字符。
- `slug` 发布后尽量不改。
- `draft: true` 的文章不得进入页面、RSS、sitemap 或搜索索引。

## 4. Markdown 安全

Markdown 渲染默认禁止原始 HTML。

安全规则：

- 禁止 `<script>`、事件属性、任意 iframe 等 HTML 注入。
- 链接只允许 `http:`、`https:`、`mailto:`。
- 禁止 `javascript:`、`data:` 等危险协议。
- MVP 不支持 MDX。
- 后续如需嵌入视频或第三方内容，用白名单短代码实现。

## 5. 动态能力边界

博客第一版保持静态优先，不做社区化功能。

策略：

- 不做评论。
- 不做点赞。
- 不展示阅读量。
- 可以接访问统计，但只供站长查看。
- 支持 RSS，不做邮件订阅。
- 后续如果需要评论，优先评估 Giscus，不优先自建评论系统。

## 6. 图片管理

图片第一版放在本地仓库，与文章放在同一目录。

规则：

- 正文图片建议小于 500KB。
- 封面图建议小于 800KB。
- 特殊图片不超过 1.5MB。
- 不直接提交未压缩大图。
- 图片必须有 alt 文案。
- 正文图片默认懒加载。
- 不推荐使用外链图片。
- MVP 不接对象存储或独立图片 CDN。

## 7. 搜索功能

MVP 做轻量本地搜索，不接第三方搜索服务。

搜索索引只包含公开文章的：

- 标题
- 摘要
- 标签
- 分类
- 日期
- URL

不索引完整正文、草稿、内部字段或未发布内容。

搜索策略：

- 点击搜索入口后再加载搜索组件和索引。
- 支持简单关键词匹配。
- 标题命中优先，其次标签、分类、摘要。
- 不做拼音搜索、错别字纠正、语义搜索。

## 8. 个人展示与作品集预留

博客需要承担个人展示作用，不只是文章归档。

MVP 阶段需要提供：

- 关于页，用于介绍个人背景、关注方向、联系方式和外部链接。
- 首页预留个人简介区域，用于展示个人定位或一句话介绍。
- 导航中预留作品集入口，但第一版不实现完整作品集管理系统。

后续作品集可扩展为独立页面，展示：

- 项目名称
- 项目简介
- 项目截图
- 技术栈
- 项目链接
- GitHub / Demo 链接
- 个人职责与成果说明

## 9. SEO 规则

SEO 信息统一由 frontmatter 生成。

每篇公开文章需要生成：

- title
- description
- canonical
- Open Graph 信息
- Twitter Card 信息

规则：

- 页面标题格式：`文章标题 | 站点名称`。
- canonical 默认由站点域名和路径生成。
- 有封面图则优先用文章封面作为 OG 图。
- 没有封面图则使用站点默认 OG 图。
- sitemap 和 RSS 只包含公开文章。
- 草稿、测试页、noindex 页面不得进入 sitemap、RSS 和搜索索引。

## 10. URL 结构

文章 URL 统一为 /posts/[slug]。
slug 发布后尽量不改。未来如需要改 slug，系统需预留 301 重定向能力（MVP 不做，但路由结构不阻塞后续实现）。

## 11. 构建性能约束

MVP 阶段文章总数 < 100 篇时，构建性能需满足：

- dev 构建（热更新）< 10s
- 生产构建 < 60s

## 12. RSS 策略

RSS 默认输出文章全文。
RSS feed 中的所有图片路径必须转为绝对 URL，确保 RSS 阅读器可正常显示。草稿和 noindex 页面不进入 RSS。

## 13. 验收标准

MVP 完成时需满足：

- 可以新增 Markdown 文章并生成页面。
- 首页、列表页、详情页、归档、标签/分类正常展示公开文章。
- 草稿不会出现在任何公开产物中。
- Markdown 渲染不会执行危险 HTML 或危险链接。
- RSS 和 sitemap 可以正常访问。
- 搜索可以按标题、摘要、标签、分类找到文章。
- 搜索索引按需加载，不影响首页首屏。
- 每篇文章有完整 frontmatter 和基础 SEO 信息。
- 首页和关于页能展示个人信息与个人定位。
- 导航或页面结构中预留作品集入口。
- 图片有 alt 文案并默认懒加载。
- 移动端阅读体验正常。

## 14. 后续可扩展方向

第一版上线后，再按需要考虑：

- Giscus 评论
- Pagefind 全文搜索
- 图片 CDN / 对象存储
- 自动图片压缩
- 动态 OG 图
- 邮件订阅
- 访问统计面板
- 相关文章推荐
